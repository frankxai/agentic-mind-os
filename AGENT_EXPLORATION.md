# How Agents Explore Agentic Mind OS

This vault is designed for agentic navigation.

## Core Navigation Map

- **Entry points**: 00_Start_Here.md, README.md, .codex/tasks.md
- **Data layers**: 01_Daily (raw), 02_Sources (distilled), 03_Concepts (atomic), 04_Patterns (recurring), 05_Learning (practice), 06_Projects, 07_Identity (canon), 08_Reviews
- **Automation**: skills/ (SKILL.md), agents/ (.agent.md), workflows/ (step-by-step playbooks)
- **Schemas**: schemas/ for validation and agent output formatting

## Example Agent Journeys

1. **Daily Capture to Canon**
   - daily-mind-capture skill → 01_Daily
   - source-distillation → 02/03
   - pattern-analyst + memory-governor → 04 + 07_Identity

2. **Weekly Reflection**
   - weekly-review skill pulls from 01/08
   - mind-cartographer updates canon
   - learning-architect proposes next experiments

## Prompt Patterns for Exploration

Use consistent invocation:
" As [Agent Name], using the human-mind models from mind-intelligence-systems, [task] on [scope]. Output in [schema]."

Agents should always:
- Reference NAMING_DOCTRINE
- Separate observation / interpretation
- Link back to models/human-mind/*
- Suggest follow-up skills/workflows

## Usefulness for Agents

- Structured folders = low hallucination
- Schemas = reliable output
- Cross-repo models = consistent self-understanding across systems
- .codex/tasks.md = persistent work queue

This makes the OS highly agent-readable while remaining human-lived.
